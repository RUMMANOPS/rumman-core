## Philosophy

n8n sits between the intelligence pipeline and human operators. Its role is **orchestration only**:
- React to events generated by platform workers
- Route information to the right output channel at the right time
- Trigger periodic maintenance and quality checks

n8n does NOT ingest messages (that's `rumman_engine.py`), does NOT embed content (that's `embed_worker.py`), and does NOT synthesize answers (that's `search_api.py`).

## Deployment Options

### Option A: Railway (Recommended)
Add n8n as a new Railway service. Connect n8n to Supabase via PostgreSQL connection, Supabase webhooks, or HTTP webhooks from platform workers.

### Option B: n8n Cloud
Managed instance at app.n8n.cloud. Simpler setup, ~$20/month. No Railway config required.

**Decision:** Start with Railway for cost control. Migrate to n8n Cloud if Railway hosting becomes a maintenance burden.
