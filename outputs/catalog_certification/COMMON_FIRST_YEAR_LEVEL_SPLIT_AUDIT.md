# Common First Year — Level 1/2 Split Audit

**التاريخ:** 2026-06-24
**النطاق:** هل يمكن عرض السنة الأولى المشتركة كمستويين (الأول/الثاني) منفصلين بدلًا من كتلة واحدة؟
**المنهج:** فحص الـ API الحي + الـ DB مباشرة (PostgREST) + المصادر الرسمية (PDF) لـ 7 برامج. **لا ذاكرة، لا تخمين.**
**القيود:** لا DB writes، لا migration، لا commit/push/deploy. لم أخترع مقررًا ولا مستوى.

---

## الخلاصة التنفيذية (اقرأ هذا أولًا — مُحدّث بدليل Banner)

المراجعة اليدوية رفضت — بحق — عرض السنة الأولى ككتلة واحدة تدمج فصلين وتخفي المستوى الثاني وتدمج ENG001 في 16 ساعة.

**تحديث جوهري (2026-06-24):** دليل تشغيلي رسمي من **Banner/Ellucian** (بيان درجات طالب) يُثبت أن السنة الأولى **مقسومة فعليًا على فصلين** كما يراها الطالب في نظام الجامعة:
> **First Term:** CI001, CS001, ENG001 (الجزء الأول)
> **Second Term 2023-2024:** COMM001, ENG001 (الجزء الثاني), MATH001

إذن التقسيم L1/L2 **موجود وحقيقي تشغيليًا** — لكن:

- **خطط الـ PDF الرسمية** تعرضه ككتلة "First Year" واحدة (لا تكشف الفصلين).
- **الكتالوج الحالي (DB/API) لا يحفظ التقسيم إطلاقًا**: لا حقل term/occurrence، و`metadata={}`، و`source_section=None`، وENG001 مدموج 16h. وترتيب الـ seq المخزّن (CI001:3, COMM001:4, CS001:5, ENG001:10/11, MATH001:18) **لا يطابق** حدود فصول Banner (COMM001 فصل ثانٍ رغم seq=4 بين مقررات الفصل الأول).

**النتيجة:** الموبايل **لا يستطيع** اشتقاق التقسيم من البيانات الحالية بثقة — اشتقاقه منها = ترتيب تعسفي/اختراع. نملك الآن مصدر الحقيقة (Banner)، لكن يجب **حقنه في الكتالوج عبر patch** ثم كشفه في الـ API.

**القرار:** `NEEDS_CATALOG_API_OCCURRENCE_PATCH_BEFORE_LEVEL_SPLIT`.

**ما يجب أن يحفظه الـ patch (من دليل Banner):** `display_level` · `academic_term_label` · `course_occurrence` · `course_part_label` · `canonical_course_code` (للربط) · `source_system='banner_ellucian'`.

**عيب إضافي مؤكَّد يُصلَح في نفس الـ patch:** دمج ENG001 (16h) → مقرران × 8h (ENG001 + ENG002 / أو occurrence 1+2).

## 1. هل source / DB / API يميز المستوى الأول والثاني؟

**لا — في كل المستويات الثلاثة.**

### المصدر الرسمي (PDF) — الدليل الحرفي

**MGT** (`study-plan-ba-dec-2025.pdf`, صفحة "Program Structure by Levels"):
```
First Year
  1 ENG001    English Language Skills              8
  2 CS001     Computer Essentials                  3
  3 COMM001   Communication Skills                 2
  4 ENG001    English Language Skills (Continued)  8
  5 MATH001   Fundamentals of Math                 3
  6 CI001     Academic Skills                      2
Level three        ← يقفز من First Year إلى Level three مباشرة
  1 ISLM101 ...
```

**DS** (`برنامج البكالوريوس في علوم البيانات.pdf`): نفس النمط — "First Year" (6 مقررات: ENG001, CS001, COMM001, **ENG002**, MATH001, CI001) → "Level three".

**IT** (`bs-it-plan-sep2023.pdf`): "First Year" → "Level three"؛ والمصدر نفسه يسرد **ENG001 = 16 ساعة كصف واحد**.

**ACC** (`...تخصص محاسبة.pdf`): "First Year" → "Level three"؛ ENG001 = 16 في جدول البنية.

**النتيجة:** الأرقام الرسمية للمستويات هي **[First Year, 3, 4, 5, 6, 7, 8]**. «Level One» و«Level Two» غير موجودين في أي مصدر. هذا تصميم أكاديمي مؤسسي لـ SEU: السنة الأولى المشتركة كتلة تأسيسية، والتخصص يبدأ رسميًا من المستوى الثالث.

### الـ DB (`cat_program_courses`)
- MGT: 5 صفوف فقط بمستوى 1، **لا صف بمستوى 2**. صف ENG001 واحد بـ `credit_hours=16`. `official_raw_text` لهذا الصف:
  `10 | ENG001 | ...Skills | 8 |  +  11 | ENG001 | ...(Continued) | 8 |` ← دمج صريح لصفين مصدريين.
- لا يوجد أي حقل `seq` / `course_order` / `semester` / `occurrence` / `part` في الجدول. الأعمدة المتاحة: level, credit_hours, category, choose_rule, official_raw_text, source_page_or_section, metadata… ولا شيء منها يحمل حدود الفصل الأول/الثاني.

### الـ API (`/programs/{code}/plan`)
- أمين للـ DB تمامًا. يعرض ENG001 صفًا واحدًا 16h، ولا يكشف `official_raw_text` ولا أي إشارة ترتيب. **لا يمكن من الـ API وحده تمييز L1/L2.**

## 2. أين تم دمج ENG001؟ (والأهم: ENG002 الحقيقي)

**اكتشاف يقلب التقرير السابق (MGT_PLAN_SEMANTICS_AUDIT):** ENG001 و **ENG002 = "English Language Skills 2"** مقرران **مستقلان** بكودين مختلفين — مُثبت من بيانات CS و DS الحية، لا من الذاكرة:

| البرنامج | تمثيل اللغة في الـ DB/API | أين حدث الدمج |
|---|---|---|
| **CS** | ENG001 (8h, L1) + **ENG002** (8h, **L2**) | لم يُدمج — سليم (لكن تقسيم L2 غير موثّق بمصدر CS؛ لم يُعثر على PDF لـ CS) |
| **DS** | ENG001 (8h) + **ENG002** (8h) — **كلاهما L1** | لم يُدمج كصف، لكن كلاهما في كتلة First Year (مطابق للمصدر) |
| **MGT** | **ENG001 = 16h** (صف وهمي واحد) | المستخرِج دمج seq10+seq11 (`ENG001` + `ENG001 Continued`) |
| **ACC/FIN/ECOM/IT** | **ENG001 = 16h** (صف وهمي واحد) | المصدر نفسه يسرد 16h في بعضها (IT/ACC) + دمج استخراج |

**الخلاصة:** الـ«16 ساعة» في MGT/ACC/FIN/ECOM/IT تمثيل مدموج. الحقيقة الأكاديمية: **مقرران × 8 ساعات** (ENG001 + ENG002، أو ENG001 + Continued). تقريري السابق «16h صحيحة، مقرر واحد بجزأين» كان **خطأً** — صحّحه هذا الفحص.

## 3. هل يمكن إصلاحه في الموبايل فقط؟

**لا — لشقّين مختلفين:**

| المطلوب | ممكن في الموبايل؟ | السبب |
|---|---|---|
| فصل ENG001 المدموج (16h) → ENG001 + ENG002 | **لا** | الـ API يعطي صفًا واحدًا 16h بلا ENG002؛ توليد ENG002 = اختراع |
| إسناد مقررات السنة الأولى لـ L1 vs L2 | **لا** | لا المصدر ولا الـ API يحملان حدّ الفصل؛ أي تقسيم = اختراع/ترتيب تعسفي |
| عرض «المستوى الأول» و«المستوى الثاني» منفصلين | **لا** | غير موجود رسميًا (انظر §1) |
| إيقاف الكتلة المدمجة المضللة وعرض ملاحظة صادقة | **نعم** | إزالة تضليل، لا اختراع — وهذا ما نُفّذ (انظر §7) |
| عرض مستويات التخصص 3–8 بشكل صحيح | **نعم** | بياناتها سليمة ومصالحة المجموع موثّقة |

## 4. هل نحتاج Catalog/API patch؟

**نعم — لكن بطبقتي ثقة مختلفتين:**

### الطبقة A — قابلة للتنفيذ من المصادر الحالية (ثقة عالية)
إصلاح دمج ENG001 → صفّين مستقلّين في `cat_program_courses`:
- `ENG001` (8h) + `ENG002` (8h) [أو `ENG001` part 2 بكود مستقل للربط]
- يحافظ على `canonical_course_code` للربط، ويضيف تمييز occurrence/part في صف العرض.
- مدعوم مباشرة من `official_raw_text` (MGT) ومن CS/DS اللذين يحملان ENG002 فعلًا.

### الطبقة B — **غير قابلة من المصادر الحالية** (تحتاج وثيقة جديدة)
إسناد مقررات السنة الأولى إلى المستوى الأول مقابل الثاني:
- **لا يمكن** اشتقاقه من خطط البنية الرسمية الحالية (كلها «First Year» ككتلة).
- يحتاج **مصدرًا رسميًا جديدًا**: الخطة الفصلية الاسترشادية للسنة الأولى / جدول الإسناد التلقائي الذي يوزّع المقررات على فصلين.
- بدون هذه الوثيقة، أي تقسيم L1/L2 = اختراع.

### حقول schema المقترحة (DRAFT — لا apply)
على `cat_program_courses`:
- `course_occurrence SMALLINT` (1/2 لـ ENG part 1/2)
- `display_level SMALLINT` (المستوى المعروض إن اختلف عن level الأكاديمي)
- `term_within_year SMALLINT` (فصل ضمن السنة — يحتاج الطبقة B)
- `course_part_label TEXT` («(Continued)» / «2»)
- `is_exemptable BOOLEAN`, `exemption_source TEXT` ('STEP'), `requires_official_verification BOOLEAN`, `does_not_count_in_gpa BOOLEAN` (كلها NULL حتى مصدر رسمي يثبتها — لا أرقام STEP الآن)
على `cat_programs`:
- `first_year_model TEXT` ('single_block' | 'two_semesters') + `academic_phase`

> **DRAFT فقط.** لا migration ولا apply. تُكتب نهائيًا عند بدء Wave B وبعد قرار §9.

## 5. ما البرامج المتأثرة؟

| البرنامج | الكلية | الحالة | النوع |
|---|---|---|---|
| MGT, ACC, FIN, ECOM | الإدارية والمالية | ENG001=16h مدموج، لا L2 | دمج + كتلة واحدة |
| IT | الحوسبة | ENG001=16h (المصدر نفسه 16h)، لا L2 | كتلة واحدة |
| DS | الحوسبة | ENG001+ENG002 منفصلان لكن كلاهما L1 | كتلة واحدة (مقرران سليمان) |
| CS | الحوسبة | ENG001(L1)+ENG002(L2) | الوحيد بـ L2 — **لكنه artifact غير موثّق بمصدر** |

**ملاحظة CS:** لم يُعثر على PDF رسمي لـ CS في المستودع؛ تقسيمه L1/L2 (L2 = ENG002 فقط) غير محقَّق بمصدر، فلا يُعتمد كمرجع.

## 6. ما الشكل الصحيح للعرض؟

نظرًا لأن المصدر الرسمي يعرّف السنة الأولى ككتلة واحدة، الشكل الصادق هو **أحد خيارين** (قرار §9):

**الخيار 1 — احترام النموذج الرسمي (متاح الآن، بلا مصدر جديد):**
```
السنة الأولى المشتركة (سنة تأسيسية — تُسند تلقائيًا)
   • التفاصيل الفصلية قيد التحقق
المستوى الثالث: [مقررات]
المستوى الرابع: [مقررات]
...
```

**الخيار 2 — التقسيم الذي طلبتَه (يحتاج وثيقة جديدة + patch):**
```
المستوى الأول — السنة الأولى المشتركة: ENG001, CS001, COMM001 ...
المستوى الثاني — السنة الأولى المشتركة: ENG002, MATH001, CI001 ...
المستوى الثالث: ...
```
**لا يمكن تنفيذ الخيار 2 الآن** لأن توزيع المقررات على الفصلين غير منشور رسميًا.

## 7. ماذا نُفّذ (موبايل فقط — مصرّح به)

أزلت الحالة المدمجة المرفوضة دون اختراع أي تقسيم:
- ملف `services/catalogPlanGate.js`: إضافة `firstYearSplitAvailable=false` لكل برامج السنة المشتركة، وحقول دلالات ENG (exemptable/STEP/verification placeholders)، وحارس Smart Registration لمنع L1/L2.
- `screens/OnboardingScreen.js`: في خريطة التقدم، تُعرض السنة الأولى ككتلة **معلوماتية صادقة** («سنة تأسيسية مشتركة، تُسند تلقائيًا، تفاصيلها الفصلية قيد التحقق») **بدون** قائمة مقررات مدمجة مضلّلة و**بدون** عرض ENG001=16h كمقرر عادي. مستويات التخصص 3–8 تُعرض كاملة وتفاعلية.
- لم أعرض «المستوى الأول/الثاني» منفصلين (غير موجود رسميًا)، ولم أخترع ENG002 في برامج الدمج.

## 8. ماذا لم أنفّذ ولماذا

- **تقسيم L1/L2 في الموبايل** — لأنه غير موجود في أي مصدر؛ تنفيذه = اختراع مستوى (محظور).
- **فصل ENG001→ENG001+ENG002 في الموبايل** — لأن الـ API يعطي صفًا مدموجًا واحدًا؛ توليد ENG002 client-side = اختراع. يلزم catalog patch.
- **أي DB write / migration / apply** — ممنوع؛ schema مقترح DRAFT فقط.
- **أرقام STEP** — لم تُعرض؛ تحتاج مصدرًا رسميًا.

## 9. القرار (مُحدّث بدليل Banner)

```
NEEDS_CATALOG_API_OCCURRENCE_PATCH_BEFORE_LEVEL_SPLIT
```

**لماذا ليس MOBILE_CAN_SPLIT:** الموبايل يقرأ من الـ API فقط، والـ API لا يحوي أي حقل term/occurrence. حتى الـ DB لا يحفظه (seq مخزّن لا يطابق حدود فصول Banner، metadata فارغ، ENG001 مدموج). أي تقسيم client-side = ترتيب تعسفي.

**مصدر الحقيقة متوفر الآن:** دليل Banner/Ellucian يحدد التقسيم بدقة:
- **First Term (المستوى الأول):** CI001, CS001, ENG001 (occurrence 1)
- **Second Term (المستوى الثاني):** COMM001, ENG001 (occurrence 2), MATH001

**الـ patch المطلوب (DRAFT — لا apply الآن):**
1. إصلاح دمج ENG001 (16h) → صفّين × 8h مع `course_occurrence` 1/2.
2. إضافة حقول: `display_level`, `academic_term_label`, `course_occurrence`, `course_part_label`, `canonical_course_code`, `source_system='banner_ellucian'`.
3. حقن توزيع الفصلين من Banner لبرامج السنة المشتركة (نموذج موحّد).
4. كشف الحقول في `/programs/{code}/plan`.
5. الموبايل يقرأ `display_level`/`academic_term_label` فيعرض «المستوى الأول — السنة الأولى المشتركة» و«المستوى الثاني — السنة الأولى المشتركة» بلا اختراع.

**الحالة الحالية للموبايل:** كتلة «السنة الأولى المشتركة» صادقة مؤقتة (لا تدّعي تقسيمًا، لا تعرض ENG001=16h مضلِّلًا) — جسر آمن حتى ينزل الـ patch، وعندها يتحول تلقائيًا لعرض الفصلين.
