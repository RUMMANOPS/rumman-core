## 2. ما سبب RTL regression؟

**ليس مشكلة اتجاه.** RTL مفروض عالميًا في `index.js` عبر `I18nManager.forceRTL(true)`، فالاتجاه حتمي (العنوان يمينًا، الـ badge يسارًا) — صحيح.

السبب الفعلي شيئان:
1. **محاذاة عمودية:** كروت البرامج كانت `alignItems: 'center'`. حين يلتف العنوان الإنجليزي الطويل إلى 3 أسطر، يطفو الـ badge في المنتصف العمودي → يبدو "مرميًّا". **الحل:** `alignItems: 'flex-start'` + `marginTop: 2` على الـ badge ليحاذي السطر الأول.
2. **هيمنة الإنجليزية:** الأسماء الإنجليزية الطويلة (16-18px bold) تلتف 3 أسطر وتسيطر بصريًا — وهذه **فجوة بيانات** (لا اسم عربي)، لا مشكلة layout.

أضفت أيضًا `writingDirection: 'rtl'` صراحةً على كل نصوص الواجهة (بدل الاعتماد الضمني على forceRTL) — حتى لا تكون المحاذاة "بالصدفة".

## 6. هل خطة MGT تظهر كما هي رسميًا أم تحتاج catalog semantics؟

**تحتاج catalog semantics — فيها 3 فجوات حقيقية:**

1. **المستوى 2 غائب تمامًا:** API يرجع المستويات `[1, 3, 4, 5, 6, 7, 8]` — لا مستوى 2. لهذا الشاشة تقفز من «مستوى 1» إلى «مستوى 3». **لم أتحقق** هل هذا لأن خطة SEU فعلًا بلا "مستوى 2" مُرقّم أم أن ingest الكتالوج أسقطه — يحتاج تحققًا مقابل PDF الخطة الرسمية الموجود في المشروع.

2. **لا عناوين عربية للمقررات:** كل `official_title_ar` = NULL.

3. **لا semantics للسنة الأولى المشتركة:** المستوى 1 يحوي مقررات السنة الأولى المشتركة (CI001, COMM001, CS001, ENG001=**16 ساعة**, MATH001 = 26 ساعة) مصنّفة `university_requirement` فقط — بلا أي إشارة إلى أنها "سنة أولى مشتركة" أو أن مقرر الإنجليزية خاص.

**المصدر الرسمي الموجود في المشروع:** `2. Regulations/ضوابط السنة الأولى المشتركة/ضوابط السنة الأولى المشتركة 2023.pdf` (سارية من 1445هـ). ينص:
- السنة الأولى المشتركة تُدرّس بالإنجليزية، مقرراتها تظهر في السجل وتدخل المعدل والتخرج.
- **استثناء: مقرر اللغة الإنجليزية تُحتسب ساعاته ضمن التخرج لكن لا يُحتسب تقديره في المعدل.**
- اجتياز السنة الأولى المشتركة شرط لإكمال الدراسة في البرنامج.

→ هذه semantics غير موجودة في Catalog API الآن.

## 7. كيف تعاملت مع English/STEP بدون تخمين؟

**لم أخمّن — وجدت المصدر الرسمي وقرأته.**

من `ضوابط السنة الأولى المشتركة 2023.pdf` صفحة 4 (سادسًا: الاختبارات) — جدول STEP/IELTS الرسمي لاجتياز مقرر اللغة الإنجليزية:

| البرنامج الأكاديمي | STEP | IELTS | ملاحظة |
|--------------------|------|-------|--------|
| القانون والإعلام الرقمي | **45** | 3 | + أعمال سنة (23) درجة في مقرر الإنجليزية |
| **جميع البرامج الأخرى عدا اللغة والترجمة** (يشمل MGT) | **60** | 4 | — |
| اللغة الإنجليزية والترجمة | **65** | 4.5 | + أعمال سنة (18) درجة |

→ **لـ MGT: STEP ≥ 60 (أو IELTS 4) يُعفي/يجتاز مقرر اللغة الإنجليزية.** مصدر موثّق، لا ذاكرة.

**ما نفّذته:** لا شيء في منطق STEP (حسب التعليمات — لا تنفيذ STEP الكامل). فقط وثّقت القاعدة + المصدر هنا.

**نموذج البيانات المقترح** (لمقررات الإنجليزية التأسيسية في الكتالوج لاحقًا):
```
is_english_foundation:        bool   // ENG001 وأمثاله
gpa_excluded:                 bool   // true — الساعات تُحتسب، التقدير لا
is_exemptable:                bool   // true عبر STEP/IELTS
```

## 9. كيف حسبت inferred_next_level؟

دالة `computeInferredNextLevel(planCourses, declaredLevel, completedCodes)` في `OnboardingScreen.js`:

```
declared_current_level = المستوى الذي اختاره الطالب (لا يُمَس)
inferred_next_level    = إشارة منفصلة مشتقة من المقررات المُعلَنة ذاتيًا
```

القواعد (مع التعامل مع المستويات غير المتصلة):
- إذا **كل** مقررات المستوى المختار مُعلَّمة مجتازة → `inferred = أصغر مستوى في الخطة أكبر من المختار`.
  - **حرج:** لـ MGT، مختار=1 وكل L1 مجتاز → التالي = **3** (يتخطى المستوى 2 الغائب)، لا 2.
- إذا الإنجاز جزئي → `inferred = declared` (يبقى).
- إذا **كل** الخطة مجتازة → `plan_complete_estimate = true` → «يبدو أنك أنهيت خطتك الدراسية تقريبًا».

**اختبارات الوحدة (5 حالات) نجحت** بما فيها الحالة غير المتصلة.

**التخزين:** الـ profile v2 يخزّن `declared_current_level` و`current_level` (=declared للتوافق) و`inferred_next_level` و`plan_complete_estimate` منفصلة — لا خلط.

**العرض في الملخص:** «بناءً على المقررات التي حدّدتها، يبدو أن مستواك القادم هو المستوى X». اختيار الطالب الأصلي يظهر منفصلًا («المستوى الذي اخترته»).

## 13. هل Wave 1 جاهزة لإعادة المراجعة اليدوية؟

**لا — ليس بعد.** الإصلاحات الـ client تمت (RTL، inferred level، wording، زر التحديث)، لكن المشاكل الأكاديمية **الأبرز بصريًا** (أسماء إنجليزية، خطة بلا مستوى 2، إنجليزية بلا semantics) **فجوات بيانات في الكتالوج**. إعادة مراجعة الموبايل الآن ستُظهر نفس الشكاوى (#2، #3) لأن المصدر هو الكتالوج لا كود الموبايل.
